How FRAME Virtual Fiscalisation works

From device registration to compliant e-receipts, FRAME Virtual Fiscalisation handles the full fiscalisation lifecycle.

1. Register devices in minutes

Connect fiscal devices, virtual devices, and channels (POS, ERP, e‑commerce) via secure APIs. ZimraFlow normalises device capabilities and abstracts away vendor quirks.

2. Issue compliant receipts

Every transaction is enriched with ZIMRA-required metadata, signed, and stored against your fiscal devices. Generate compliant PDFs, QR codes, and audit trails automatically.

3. Automate ZIMRA reporting

Stream fiscal data to your reporting stack or download ready-made exports. Alerts, reconciliation dashboards, and exception handling keep your team ahead of audits.

Developer & API hub

Modern, well-documented APIs that make fiscalisation just another service call.

  • Language-agnostic REST APIs with JSON payloads
  • Test keys, sandbox mode, and predictable versioning
  • Webhook callbacks for device and receipt lifecycle events

Use the sample below to register a device and send a fiscal receipt. Then trigger the mock API demo to see a JSON response.

// 1. Register a fiscal device
fetch("https://api.zimraflow.io/v1/devices", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    deviceId: "ZIMRA-HQ-001",
    channel: "POS",
    branch: "Harare HQ"
  })
});

// 2. Issue a fiscal receipt
fetch("https://api.zimraflow.io/v1/receipts", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    deviceId: "ZIMRA-HQ-001",
    receiptNumber: "INV-24801",
    currency: "ZWL",
    totalAmount: 1250.00,
    taxAmount: 187.50,
    items: [{ sku: "SKU-001", description: "Smart POS", qty: 1, unitPrice: 1250.00 }]
  })
});
